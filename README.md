# LoraBridge Messenger

LoraBridge — это мост между Meshtastic и Telegram. Он позволяет операторам LoRa‑сетей общаться с командой в мессенджере, не
покидая диапазон. Радиоузел подключается по USB, выбирается нужный COM‑порт, а бот в Telegram принимает и отправляет сообщения.
Проект создан для команд, которым важно держать связь в офлайне: горные экспедиции, службы реагирования и полевые испытания.

## Что уже готово
- Очереди доставки, Telegram/Meshtastic клиенты и менеджер соответствия ID.
- CLI (`python -m src.main`) c командами `run`, `status`, `flush`.
- Логи и пользовательские базы в `lorabridge/data`.
- Подробная документация (`lorabridge/docs/README_RU.md`, `docs/wiki/*`).

## Что появится дальше

- Автоматическая диагностика LoRa‑сети и интеграция с новыми типами узлов.

## Как запустить
1. Установите Python 3.11+ и Git.
2. Клонируйте репозиторий и перейдите в проект:
   ```bash
   git clone <repo-url>
   cd MESHGRAM/lorabridge
   python -m venv .venv
   source .venv/bin/activate  # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```
3. Запустите любую команду (`run` или `status`). Перед стартом откроется новый мастер настройки:
   ```bash
   python -m src.main run
   ```
   Мастер попросит войти в Telegram (ввести токен бота прямо в консоль), выбрать COM‑порт из найденных устройств, задать скорость,
   интервал дросселирования и прочие параметры. Всё сохраняется автоматически в `lorabridge/config/.env` и
   `lorabridge/config/settings.json`, поэтому больше не нужно выполнять `echo "TELEGRAM_TOKEN=..."` или редактировать JSON
   вручную. При повторном запуске мастер предложит оставить текущие настройки или перенастроить мост.

Дальнейшие сценарии (`status`, `flush`, сборка через PyInstaller) описаны в `lorabridge/README.md` и `lorabridge/docs/wiki`.
