# Meshgram

Личный мост между Telegram и Meshtastic LoRa-сетью. До тебя можно дозвониться даже когда у тебя нет интернета — пока карманная нода держит mesh-сигнал.

[meshgram.site](https://meshgram.site) · [старый прототип (ноябрь 2025)](https://github.com/XYZ-WEB3/MESHGRAM/tree/old)

## Архитектура

```
TG-юзер → Telegram-бот → home-нода (USB) ──LoRa DM──► карманная нода
                                                              ↓
                                                      читаешь, отвечаешь
                                                              ↓
TG-юзер ← Telegram-бот ← home-нода ──LoRa DM──◄ карманная нода
```

Каждой переписке в Telegram назначается короткий слот `@N`. Ответ, адресованный этому слоту, уходит обратно конкретному отправителю — никакого общего чата, никакой широковещалки. Между двумя нодами идёт зашифрованный DM, чужие узлы канала только ретранслируют пакет.

## Возможности

- **Sticky-слоты `@N`** — один номер закреплён за человеком на серии сообщений. Экран карманной ноды ведёт себя предсказуемо.
- **Статусы доставки** как в обычном мессенджере: *передаю → отправлено → доставлено → ответ*.
- **Очередь авто-повторов** — если pocket вне связи, исходящие сохраняются в SQLite и отправляются позже. Переживает рестарт процесса.
- **Категории и реферальные ссылки** — `t.me/bot?start=work`, `?start=family`. Префикс категории показан рядом с отправителем — контекст очевиден.
- **GPS через `/where`** (бета) — координаты доступны только избранным контактам.
- **SOS** — `#SOS <текст>` с карманной ноды веером рассылается списку получателей, опционально с координатами.
- **Whitelist** — закрытый режим, бан-лист, избранные.
- **PyQt6 desktop-клиент** — настройки, пользователи, активные слоты, live-лог релея, селектор из 50 моделей нод.
- **Чанкинг длинных сообщений** — текст, не вмещающийся в LoRa-кадр, автоматически разбивается на части `[1/N] … [N/N]`.
- **Переключатель скорость/надёжность** — `MESH_DELIVERY_MODE=fast` для срочных коротких сообщений (fire-and-forget), `reliable` (по умолчанию) — со статусами и ретраями.
- **SOS fast-retry** — сообщения с `#SOS`, `срочно` или `urgent` ретраятся с шагом 5/15/30/60/120 секунд вместо обычных 2–15 минут.
- **AI-помощник** (опционально, по умолчанию выключен) — с карманной ноды пишешь `@ai <вопрос>`, локальная LLM отвечает. Продолжение диалога — `@aiN <вопрос>`. Работает с любым OpenAI-совместимым endpoint'ом; дефолтный конфиг — [LM Studio](https://lmstudio.ai/) на `localhost:1234`, переписка не покидает устройство. Системный промпт и TTL настраиваются.

## Установка

Нужно:

- Python 3.10 или новее
- Две Meshtastic-совместимые LoRa-ноды
- Telegram-бот из [@BotFather](https://t.me/BotFather) и твой numeric Telegram ID из [@my_id_bot](https://t.me/my_id_bot)

```bash
git clone https://github.com/XYZ-WEB3/MESHGRAM.git
cd MESHGRAM
pip install -r requirements.txt
```

### Windows (GUI)

Двойной клик по `relay\run_gui.bat`. На первом запуске мастер собирает токен бота, твой Telegram ID, ID карманной ноды и COM-порт.

### Windows (CLI) / macOS / Linux

```bash
cp relay/.env.example relay/.env
# отредактировать .env: BOT_TOKEN, OWNER_ID, POCKET_NODE_ID
cd relay
python relay.py --port COM3          # Windows
python relay.py --port /dev/ttyUSB0   # Linux / macOS
```

Для удобства приложен bash-лаунчер `relay/run_relay.sh` — он создаёт `venv`, ставит зависимости, листит serial-устройства. Production-инструкция для Linux с `systemd`-юнитом — [`relay/deploy/INSTALL_LINUX.md`](relay/deploy/INSTALL_LINUX.md).

## Статус проекта

### Сделано

- **Слот-роутинг v1** — sticky `@N`, заменивший прежний broadcast-мост
- **ACK по LoRa** — статусы доставки на основе wantAck, retry-очередь с экспоненциальным backoff'ом
- **GPS / `/where` (бета)**, SOS, whitelist, категории
- **PyQt6 desktop-клиент** с селектором 50 моделей нод
- **meshgram.site** — лендинг (Caddy + Let's Encrypt)
- **Latency batch** — TG → pocket ускорен с 10–15 с до примерно 3 с (см. [CHANGES.md](CHANGES.md))
- **Linux-поддержка** — `.sh`-лаунчеры, `systemd`-юнит, install-guide
- **UTF-8 chunking** длинных сообщений, переключатель скорость/надёжность, fast-retry для срочных

### В работе

- **Cloud-режим** — облачный вариант: пользователь держит только Wi-Fi-ноду дома, relay работает на инфраструктуре проекта. E2E-тесты пройдены, идёт **закрытая бета на 25 слотов** — счётчик и форма ожидания на [meshgram.site](https://meshgram.site). Открытая регистрация будет анонсирована отдельно после стабилизации.

### В планах

- **Multi-user** — один процесс обслуживает несколько ботов через общую gateway-ноду
- **Карта в GUI** — Leaflet/OSM с последним GPS-фиксом карманной ноды
- **Управление нодой из GUI** — мощность LoRa, роль, регион, рестарт
- **Сеть ROUTER-нод** — гайд по разворачиванию ретрансляторов с городским покрытием
- **Готовые пакеты** — `.deb` для Debian/Ubuntu, AUR для Arch, PyInstaller-бандл для Windows
- **Локализация** — английская и испанская версии сайта, бота и CLI

### Рассматривали и отложили

- **MeshCore как второй mesh-протокол** — перспективный новый LoRa-mesh-стек с детерминированной source-routed маршрутизацией (эффективнее по эфирному времени, чем flooding в Meshtastic), с официальной Python-библиотекой и встроенными Serial / TCP / BLE-транспортами. Рассматривали для v0.8 с переключателем `MESH_PROTOCOL=meshtastic|meshcore` и абстракцией `MeshInterface`. Отложили: в официальном списке регионов MeshCore нет России, нет нативного мобильного клиента уровня Meshtastic (iOS/Android), и в нашем регионе развёрнутых MeshCore-нод слишком мало — пользователь, поднимающий Meshgram поверх MeshCore, просто не с кем будет переписываться. Вернёмся к идее, когда проект наберёт массу в РФ и появится клиент под смартфон.

Подробный roadmap с разбивкой по задачам — в [CHANGES.md](CHANGES.md). Архитектура и справочник команд — в [relay/README.md](relay/README.md).

## Стек

Python 3.10+, PyQt6, python-telegram-bot, meshtastic-python, SQLite (WAL), Caddy + Let's Encrypt для сайта.

## Кредиты

- [Meshtastic](https://meshtastic.org/) — open-source LoRa-mesh, на котором всё работает
- SVG-картинки моделей нод в [`relay/devices/`](relay/devices) — с [официальной документации Meshtastic](https://meshtastic.org/docs/hardware/devices/)
- Иконки UI следуют стилю Lucide / Feather

Проект не аффилирован с Meshtastic, использует только публичный SDK и открытые ассеты.
